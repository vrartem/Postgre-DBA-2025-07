## Курсовая работа на тему: "Разработка геоинформационной системы на базе PostgreSQL / PostGIS"???

На сельскохозяйственной технике установлено программно‑аппаратное оборудование, которое в процессе выполнения работ (сбор урожая, посев, внесение удобрений и т. п.) формирует шейп‑файлы (.shp) в режиме реального времени.
В этих файлах с высокой точностью фиксируются:
	- геополигоны, отражающие пройденные участки поля;
	- данные полигона (время выполнения, скорость движения, объём внесённых материалов, урожайность, химический анализ почвы и др.).
При большем количестве техники и разнообразии работ возникает необходимость иметь систему по сбору, хранению и анализу данной информации.

### Что будет использовано
* Ubuntu 24.04.3
* PostgreSQL 16.10 (Ubuntu 16.10-0ubuntu0.24.04.1)
* Расширение PostGIS 3.4.2
* shp2pgsql
* DBeaver 25.2.2
* QGIS

### Что будет выполнено:

1. Установка кластера postgres, нагрузочное тестирование и настройка, установка расширения PostGIS, создание пользователей, схем, доступов...
2. Создание на сервере папки с sftp доступом.
3. Создание триггера, который будет импортировать данные в postgres из архива с шейп-файлами, как только появится новый архив в папке sftp.
	Буду использовать shp2pgsql — это утилита командной строки для импорта шейп-файлов в базу данных PostgreSQL/PostGIS, входит в стандартный комплект поставки расширения PostGIS. 
	Она конвертирует геоданные из формата .shp в SQL-запросы, которые затем выполняются для загрузки в БД.
4. Использование пространственных индексов, проверка план запросов, сравнение разницы.
5. Использование функций postgis, например, ST_Intersects, ST_Area, ST_Length, ST_Distance и др
5. Аналитика, создание запросов для построения карты урожайности, карты скорости обработки, подсчет периметра, процент наложения.
7. Работа с итоговой базой в dbeaver, qgis
