- Postgresql. PostGIS
- Создание базы для работы с геоданными

Установлено специальное программно-аппаратное оборудование на технике которая выполняет различные сельхоз работы (сбор урожая, сев, внесение удобрение...)
Устройства создают шейп-файлы (.shp) в которых сохраняются геополигоны и их атрибуты с сантиметровой точностью. Материалы реальны, задача актуальна.
Необходимо реальзовать сбор, хранение и налализ полученных данных на основе базы данных postgresql. 

- Что будет выполнено:

1. Установка кластера postgres, нагрузочное тестирование и настройка, установка расширения PostGIS, создание пользователей, схем, доступов...
2. Создание на сервере папки с sftp доступом.
3. Создание триггер который будет импортировать данные в postgres из архива с шейп-файлами как только появится новый архив в папке sftp.
	Буду использовать shp2pgsql — это утилита командной строки для импорта шейп-файлов в базу данных PostgreSQL/PostGIS. 
	Она конвертирует геоданные из формата .shp в SQL-запросы, которые затем выполняются для загрузки в БД.
4. Наложение пространственных индексов, проверка как строится план запросов, сравнить разницу.
5. Аналитика, создание запросов для построения карты урожайности, карты скорости обработки, подсчет периметра, процент наложения.
6. работа с итоговой базой в dbeaver, qgis
